Combover.JS

Markup, it's like hair you don't need… just Combover

Combover removes the need for extra markup by binding objects directly to HTML tags. It’s simple, lightweight and intuitive… everything a Javascript templating engine should be.

We find that our HTML is usually in the structure of the data we want to template. So why have additional markup surrounding our HTML? It breaks HTML renders, WYSIYWGS and look's ugly.

Getting Started

Combover is a little different then most templating engines. Instead of additional markup tags you simply tell Combover which objects bind to your HTML tags using the “comb” attribute.

JSON object

{
 'name':'Hair to Stay',
 'phone':'212-219-3416'
}

HTML Template

<ul>
 <li>Name:<span comb='name'></span></li>  <li>Phone:<span comb='phone'><span></li> </ul>

Result

<ul>
 <li>Name:<span>Hair to Stay</span></li>  <li>Phone:<span>212-219-3416<span></li>
</ul>
Templates

Combover templates work just like other engines. You can send in HTML source or supply an element ID

 var gutsy = Combover.compile('#rusk-gutsy');

You can also use the script tag to surround your templates or pull them in externally

<script id='rusk-gutsy' type='text/x-combover-template'></script> Using a Template

Once you have a compiled template just send it the data. The compile method returns a rending function that you simply send in data and out the other side you get the HTML.

var gutsy = Combover.compile('#rusk-gutsy');
var data = {name: 'Hair to Stay', phone: '212-219-3416'};
var html = gutsy(data);

Templating

A template is simply HTML. By adding the "comb" data attributes to tags you are telling Combover you want to bind your content to it.

Templates can be

  • Inline - template is in the body of your HTML
  • Text
  • Text via <script> tag

The attributes can be either "comb" or the HTML5 data attribute "data-comb"

Why "comb"and "data-comb"? "comb" is simple and shorter and we like simple & short.

use the HTML5 data attribute "data-comb" when using Inline templates so HTML5 validators will not freak out. It’s also completely safe to use the shorthand attribute "comb" when using script or text templates

The "comb" and "data-comb" attributes will be removed in the rendered output

The Combover attribute value is comprised of 3 components

  • Renders
  • Helpers
  • Object property name

We'll get to Renders & Helpers in a minute. The most important part is the object property name. You are just mapping your object to the HTML tags. The tag then becomes the object value and any child tags are now in that values context. When the value is an object (nested objects) the context is this object.

JSON object

{
gusty: {
 'brand':'RUSK',
 'name':'Being Gutsy',
 'type':'Thickening Cream',
 'price':9.95
},
wild: {
 'brand':'RUSK',
 'name':'Being Wild',
 'type':'Paste',
 'price':8.95
},
}

HTML Template

<div comb='gusty'>
 <div comb='name'></div>
 <div comb='price'></div>
</div>

Result

<div>
 Name:<div comb="name">Being Gutsy</div>
 Phone:<div comb="phone">9.95<div>
</div>

Renderers

Renders are methods to tell Combover to render that particular tag. Renders precede the property name.

! value must not === false so no:

  • empty strings
  • 0
  • false
  • null or undefined

? Value is set. Meaning it is not undefined

JSON object

{
gusty: {
 'brand':'RUSK',
 'name':'Being Gutsy',
 'type':'Thickening Cream',
 'price':9.95,
 'in_stock': false
},
wild: {
 'brand':'RUSK',
 'name':'Being Wild',
 'type':'Paste',
 'price':8.95
 'in_stock': true
},
}

HTML Template

<div comb="gusty">
 <div comb="name"></div>
 <div comb="price"></div>
 <div comb="!in_stock">OUT OF STOCK</div>
</div>
<div comb="wild">
 <div comb="name"></div>
 <div comb="price"></div>
 <div comb="!in_stock">OUT OF STOCK</div>
</div>

Result

<div>
 <div>Being Gutsy</div>
 <div>9.95<div>
 <div>OUT OF STOCK</div>
</div>
<div>
 <div>Being Wild</div>
 <div>9.95<div>
</div>
Customer Renderers

You can create your own renders using the addRenderer method which takes a symbol and a callback

Combover.addRenderer('~', function(val) { return val == 'Bald' });

Symbol - Must be a special character

Callback - Takes the current value and returns TRUE if you want to render that element

Helpers

Helpers are methods that format the output. They are placed after the object property

  • ∧ — Uppercase
  • $ — Format to a currency with 2 decimal places "9.9512" -> $9.95
  • % — Formats to percentage ".102" -> 10%
  • # — Formats to a number with 2 decimals places
  • , — Formats to a number with 0 decimal places

JSON object

{
 'brand':'RUSK',
 'name':'Being Gutsy'
}

HTML Template

<div comb="name^"></div>
<div comb="price$"></div>

Result

<div>BEING GUTSY</div>
<div>$9.95<div>
Custom Helpers

Use the addHelper method which takes a symbol and a callback

Combover.addRenderer('@', function(val) {
 return val.replace('@', 'at').replace('.', 'dot');
});

Symbol - Must be a special character

Callback - Takes the current value and returns the output
